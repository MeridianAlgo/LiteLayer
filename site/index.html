Self-hosted NAS backend · Raspberry Pi

Plug in a drive.
Browse it anywhere.

LiteLayer turns a Raspberry Pi and any USB drive into a private file server you reach from any device — over your LAN or any VPN. It mounts read-only by default and never erases what's already there.

$ bash <(curl -fsSL litelayer/install.sh)

One line over SSH. Pi 3 → Pi 5 · 32 & 64-bit · ~5 min.

What it is
ANY FS
Reads every drive
ext4, NTFS, exFAT, FAT32, Btrfs, XFS, HFS+, F2FS — kernel auto-detect as the fallback.
SAFE
Read-only first
Write is an explicit per-drive opt-in. No mkfs, no fdisk, no parted — ever.
REACHABLE
Any device, any VPN
Browse from a phone or laptop over LAN, Tailscale, WireGuard, ZeroTier — no app changes.
SELF-RUN
Updates itself
OTA over GitHub with a daily check and one-tap apply from the web UI.
Under the hood

A small, honest stack that gets out of the way.

FastAPI does the work; Caddy fronts it with local TLS; the kernel does the mounting. Nothing exotic to maintain.

hotplug

Detects on plug-in

pyudev watches for new drives with a polling fallback, so a freshly inserted disk shows up on its own.

confinement

No path escapes

Every request path is realpath-resolved and confined to the mount root. .. traversal and symlink escapes return 403.

transfer

Drive-to-drive copy

Move files between disks in the background with a live progress bar — write is auto-enabled only on the destination.

viewer

View, don't download

Open images right in the browser instead of forcing a download for every file you want to glance at.

auth

Locked by default

argon2 password hashing, HttpOnly session cookie or Bearer token. Every drive and file endpoint requires a session.

mobile

Works on a phone

A two-screen layout for small screens — drives are home, tap one to browse its files. No off-screen overflow.

Safety rules

Built so a NAS can't wreck your data.

These aren't settings you can fumble — they're how LiteLayer behaves out of the box.

Read-only by default
Read-write is an explicit per-drive opt-in via POST /api/drives/{id}/enable-write.
System disk protected
The boot disk is found via findmnt / and excluded — it is never mounted or touched.
Path confinement
Every path is realpath-resolved; .. traversal and symlink escapes return 403.
No formatting
mount.py never calls mkfs, fdisk, or parted. There is no code path that erases a drive.
Soft eject
Unmount runs sync before umount, so an eject never loses in-flight writes.
Compatibility

Runs on the Pi you already have.

Pi 3 through Pi 5, 1 GB and up, 32- or 64-bit Raspberry Pi OS (Bullseye+, Bookworm recommended). On a Pi Zero 2 W it drops to a single worker. NVMe HATs work on Pi 5.

Get started

SSH into your Pi. Run one line.

The installer sets a password, picks a VPN (or none), and registers a systemd service so LiteLayer starts on every boot. Then open http://litelayer.local.

$ bash <(curl -fsSL litelayer/install.sh)

Full guide & API reference in the repository.
